# Job Application Repository — Rules & Conventions

## Context
This repository manages LaTeX resumes and cover letters for AI Engineer job applications in Germany.
Always follow these rules exactly.

---

## Mandatory Workflow for Each New Application

1. **Choose category and create the application folder**
    - Every new application must be created under exactly one category folder:
      - `applications/full-time/`
      - `applications/master-thesis/`
      - `applications/phd/`
      - `applications/werk-student/`
      - `applications/internship/`
    - Create a new job-specific folder inside that category first. Make the folder name descriptive to avoid duplicates (e.g., `bmw-master-thesis-benchmarking-generative-ai` instead of just `bmw-generative-ai`).
    - Copy **only** these two source files from `master/` into that folder:
       - `master/master-resume.tex`
       - `master/master-cover-letter.tex`
    - Do **not** copy any other files or folders (no PDFs, no `.aux/.fdb_latexmk/.fls/.synctex.gz`, no `templates/`, no other artifacts).
    - Example commands:
       ```bash
       mkdir -p applications/<category>/<company-role>
       cp master/master-resume.tex applications/<category>/<company-role>/
       cp master/master-cover-letter.tex applications/<category>/<company-role>/
       ```
       ```powershell
       New-Item -ItemType Directory -Path "applications/<category>/<company-role>" -Force
       Copy-Item "master/master-resume.tex" "applications/<category>/<company-role>/"
       Copy-Item "master/master-cover-letter.tex" "applications/<category>/<company-role>/"
       ```
2. **Rename template files**
   - In the new folder, rename:
     - `master-resume.tex` → `[your_name]_resume.tex`
     - `master-cover-letter.tex` → `[your_name]_cover-letter.tex`
3. **Create notes file**
   - Immediately create `notes.md` in the job folder.
4. **Paste job description**
   - Include the full job description text in `notes.md`.
5. **Add generation metadata**
   - Record sources, edited sections, layout constraints, and research notes in `notes.md`.
6. **Tailor resume content**
   - **Source of Truth:** Refer to `about_me.md` to find relevant skills, background context, and experiences to use for tailoring.
   - Update only the allowed sections (`Role` line under the name header, `Profile`, `Professional Experience`, `Skills`) to align with the job description.
   - Change the `Role` line only when the job description clearly supports a better-aligned title.
   - Use keywords, tools, responsibilities, and domain language from the JD.
7. **Edit cover letter**
   - Rewrite the content fully for the target company and role, drawing upon specific details and achievements from `about_me.md`.
   - Maintain the original LaTeX layout and format.
   - Ensure body is exactly two paragraphs and under one page.

## Category Routing Rules (Mandatory)

Route each new job to the correct `applications/` category before creating files:

1. **master-thesis**
   - Use for roles containing terms such as: master thesis, thesis, masterarbeit.

2. **phd**
   - Use for roles containing terms such as: PhD, doctoral, doctorate, doktorand.

3. **werk-student**
   - Use for roles containing terms such as: working student, werkstudent, student job.

4. **internship**
   - Use for roles containing terms such as: intern, internship, praktikant, praktikum.

5. **full-time**
   - Default category for all standard permanent/non-student roles when none of the above apply.

6. **No uncategorized job folders**
   - Do not create direct job folders under `applications/` root.
   - Job folders must exist only as `applications/<category>/<company-role>/`.

## `notes.md` Required Format

The `notes.md` file must be structured to capture both metadata and full job-related context, including any questions from the JD or application portal.

1. **First line metadata** (required):
   - Include Date, Company, Role, Location, and Link  
   Example:  
   # 2026-03-16 | Company: Bosch | Role: Data Analyst | Location: Germany | Link: <job-link>

2. **Questionnaire / Application Details** (if any):
    - Answer **only** the questions explicitly present in:
       - the job description text,
       - the application portal form text,
       - and any additional text pasted by the user.
    - Do **not** generate a fixed/default questionnaire for every application.
    - If no questions are provided in the JD/portal/user text, add: `No explicit application questions were provided.`
    - Keep answers concise, specific, and based on actual experience.
    - Include direct links (LinkedIn/Portfolio) only when asked by the JD/portal/user text.

    - Generate a LinkedIn Easy Apply **“Mark this job as a top choice”** message (under **250 words**) **only if** the application is clearly LinkedIn Easy Apply and that field exists. Otherwise, do not generate this section.

3. **Job Description**:
   - Paste the full job description text exactly as provided in the JD.

4. **Generation Details**:
   - Resume source: master/master-resume.tex
   - Cover letter source: master/master-cover-letter.tex
   - Resume sections edited: Profile, Professional Experience, Skills
   - Resume constraints followed: layout unchanged, section scope unchanged
   - Cover letter constraints followed: 2 paragraphs, under 1 page, human tone
   - Research summary used: company website + public company/about information

## Resume Editing Rules (Strict, JD-Focused)

When editing `[your_name]_resume.tex`, focus only on these sections:
- `Role` line in the title block (e.g., `\large{...}` under the candidate name)
- `Profile`
- `Professional Experience`
- `Skills`

Rules for editing:

- Rewrite the content within allowed sections **fully** to match the job description, including:
   - Role title wording in the header (only if JD alignment justifies a change)
  - Summary/Profile wording
  - Job experience bullet points
  - Skills and skill headings
- Prioritize JD keywords, required tools, stack, responsibilities, and domain language.
- Apply explicit length limits:
   - Profile summary must not exceed 330 characters total (including spaces).
   - Professional Experience must stay in bullet-point format for each job role.
   - Each Professional Experience point must not exceed 150 characters (including spaces).
   - If a point exceeds 150 characters, rewrite it to be shorter while keeping the same meaning.
- Maintain the **existing LaTeX structure**:
  - Do not add or remove sections
  - Do not add or remove bullet points (you may **reword existing bullets** for relevance)
  - Keep section order unchanged
  - Keep line spacing, layout, and formatting consistent
  - Do not modify fonts, margins, geometry, or package settings

Goals:

- Make the resume **highly relevant to the JD** while keeping the PDF layout unchanged.
- Adjust wording, phrasing, headings, and (when appropriate) the role title to reflect skills, technologies, and responsibilities listed in the JD.
- Ensure the final resume communicates expertise and aligns with the target role in both wording and style.

## Cover Letter Rules (Strict)

When editing `[your_name]_cover-letter.tex`:
- Preserve the original LaTeX format/layout.
- Rewrite the textual content fully for the target company and role.
- Keep it natural and human in tone.
- Include signals of company research from public online information.
- Keep it under one page.
- Use exactly 2 paragraphs for the body content.
- Do not use `--` symbols.
- Do not include AI-identification phrases or markers.

## Experience Framing Rules by Application Type (Mandatory)

Use the current Bosch experience wording based on role type:

1. **Full-time roles**
   - In resume, cover letter, and application email, explicitly state that you are **currently working at Robert Bosch** on **Gen AI and AI workflows**, including **RAG pipelines** and **agentic workflows**.
   - Prefer present-tense phrasing (e.g., "Currently working at Robert Bosch...").

2. **Student roles (Master thesis / Working Student / Internship)**
   - In resume, cover letter, and application email, explicitly state that you are a **Master's student** and **currently working as a Werkstudent at Robert Bosch**.
   - Keep both parts together where possible (student status + Bosch current role).

3. **Consistency requirement**
   - Ensure this framing is consistent across `[your_name]_resume.tex`, `[your_name]_cover-letter.tex`, `notes.md` questionnaire answers (if relevant), and outbound email text.
   - Do not contradict timeline wording across documents for the same application.

## Tracking Rules

- Always add one row at the top in `tracker.csv` for each application.
- Use `DD-MM-YYYY` for the date and add to column `Date` the application date (default todays date) .
- Use `Application Type` column for filterable categories such as `Full-time`, `Student Job`, `Thesis`, `PhD`, or `Contract`.
- Keep status flow: `To Apply` → `Applied` → `Forwarded` or `Rejected`  → `Interview` → `Offer` → `Rejected` → `Accepted`. (Default keep as Applied)

---

## Build & Git Rules

- Do not perform any automatic compilation, CI checks, or deletion of files; these actions are prohibited to avoid extra loops and delays.
- Never edit an application folder after submission (snapshot rule).

## Application Editing Instructions

When preparing a new application:

- Always start from the `master/` folder and copy **only** `master-resume.tex` and `master-cover-letter.tex` into the job-specific folder.
- Always create the job folder in a category path first: `applications/<category>/<company-role>/`. Ensure `<company-role>` is descriptive to prevent duplicates (e.g., `bmw-master-thesis-benchmarking-generative-ai`).
- Never create uncategorized folders directly under `applications/`.
- Never copy any generated or unrelated files/folders (PDFs, LaTeX meta files, `templates/`, or other artifacts).
- Preferred explicit copy sequence:
   - Create folder: `applications/<category>/<company-role>/`
   - Copy: `master/master-resume.tex` and `master/master-cover-letter.tex`
- Rename the template files in the new folder:
  - `master-resume.tex` → `[your_name]_resume.tex`
  - `master-cover-letter.tex` → `[your_name]_cover-letter.tex`
- Create `notes.md` immediately, including:
  - First-line metadata (date, company, role, location, link)
  - Full job description
   - Question-specific answers derived only from JD/portal/user-provided text (no default fixed Q\&A)
   - LinkedIn Easy Apply “Top Choice” message only when the posting is LinkedIn Easy Apply and explicitly requests it
- Update resume content **only in the allowed sections**:
   - `Role` line under the candidate name (only when JD-based alignment makes sense)
  - `Profile`
  - `Professional Experience`
  - `Skills`
- Keep resume layout, formatting, and structure unchanged.
- Rewrite the cover letter fully for the target company and role:
  - Preserve the original LaTeX layout
  - Use exactly **two body paragraphs**
  - Keep it **under one page**
  - Maintain a professional, natural tone

---

## Dynamic Question Handling (Mandatory)

- `notes.md` must be driven by the actual input context for that application.
- Source of truth for questions: JD text + portal questions + user-pasted instructions.
- Never reuse unrelated fixed questions from previous applications.
- If a question exists, provide a direct answer under a clearly labeled heading in `notes.md`.
- If no questions exist, explicitly note that none were provided.
