# Job Application Repository — Rules & Conventions
## Context
This repository manages LaTeX resumes and cover letters for [YOUR_ROLE] job applications in [YOUR_COUNTRY/REGION].
Always follow these rules exactly when generating a new application.

## Mandatory Workflow for Each New Application
1. **Choose category and create the application folder**
    - Every new application must be created under exactly one category folder (e.g., `applications/full-time/`).
    - Create a new job-specific folder inside that category first. Make the folder name descriptive (e.g., `google-software-engineer-frontend`).
    - Copy **only** these two source files from `master/` into that folder:
       - `master/master-resume.tex`
       - `master/master-cover-letter.tex`
    - Do **not** copy any other files or folders.
2. **Rename template files**
   - In the new folder, rename:
     - `master-resume.tex` → `[YOUR_FIRSTNAME]_resume.tex`
     - `master-cover-letter.tex` → `[YOUR_FIRSTNAME]_cover-letter.tex`
3. **Create notes file**
   - Immediately create `notes.md` in the job folder.
4. **Paste job description**
   - Include the full job description text in `notes.md`.
5. **Tailor resume content**
   - Update only the allowed sections (e.g., `Profile`, `Professional Experience`, `Skills`) to align with the job description.
   - Use keywords, tools, responsibilities, and domain language from the JD.
6. **Edit cover letter**
   - Rewrite the content fully for the target company and role.
   - Maintain the original LaTeX layout and format.
   - Ensure body is exactly two paragraphs and under one page.
---
## `notes.md` Required Format
The `notes.md` file must be structured to capture both metadata and full job-related context.
1. **First line metadata** (required):
   - Include Date, Company, Role, Location, and Link  
   Example:  
   `# 2026-03-16 | Company: Google | Role: Frontend Engineer | Location: Remote | Link: <job-link>`
2. **Questionnaire / Application Details** (if any):
    - Answer **only** the questions explicitly present in the job description text or application portal.
    - Keep answers concise, specific, and based on actual experience.
3. **Job Description**:
   - Paste the full job description text exactly as provided in the JD.
---
## Resume Editing Rules (Strict, JD-Focused)
When editing `[YOUR_FIRSTNAME]_resume.tex`, focus only on these sections:
- `Role` line in the title block 
- `Profile`
- `Professional Experience`
- `Skills`
Rules for editing:
- Rewrite the content within allowed sections **fully** to match the job description.
- Prioritize JD keywords, required tools, stack, responsibilities, and domain language.
- Maintain the **existing LaTeX structure**:
  - Do not add or remove sections or bullet points (reword existing ones instead).
  - Do not modify fonts, margins, geometry, or package settings.
---
## Cover Letter Rules (Strict)
When editing `[YOUR_FIRSTNAME]_cover-letter.tex`:
- Preserve the original LaTeX format/layout.
- Rewrite the textual content fully for the target company and role.
- Keep it natural and human in tone.
- Include signals of company research from public online information.
- Keep it under one page.
---
## Tracking Rules
- Always add one row at the top in `tracker.csv` for each application.
- Use `DD-MM-YYYY` for the date.
- Keep status flow: `To Apply` → `Applied` → `Interview` → `Offer` → `Rejected` → `Accepted`. 
---
## Build & Git Rules
- Do not perform any automatic compilation or CI checks; these actions are prohibited to avoid extra loops and delays.
- Never edit an application folder after submission.
